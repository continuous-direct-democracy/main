# Сравнение с Liquid Democracy

## Что такое Liquid Democracy

**Liquid Democracy** (жидкая демократия) — это модель принятия решений, которая сочетает элементы прямой и представительной демократии. Гражданин может либо голосовать лично, либо делегировать свой голос другому человеку. Наиболее известные реализации — [LiquidFeedback](https://liquidfeedback.org/) и [Decidim](https://decidim.org/).

На первый взгляд, предлагаемая нами система похожа на Liquid Democracy. Однако между ними есть **принципиальные отличия**.

## Что общего

Обе системы разделяют несколько базовых принципов:

- **Делегирование голоса** — гражданин может передать свой голос доверенному лицу
- **Override (перехват)** — гражданин может проголосовать лично по любому вопросу, даже если делегировал голос
- **Прямое участие** — каждый гражданин имеет право голосовать по любому вопросу напрямую

## Ключевые отличия

### 1. Цепочки делегирования

| Аспект                     | Liquid Democracy | Наша система     |
| -------------------------- | ---------------- | ---------------- |
| Транзитивное делегирование | ✅ Да            | ❌ Нет           |
| Уровни делегирования       | Неограниченно    | Только 1 уровень |
| Риск «суперголосователей»  | Высокий          | Исключён         |

**Liquid Democracy** поддерживает **транзитивное делегирование**: если Анна делегировала Борису, а Борис делегировал Виктору, то Виктор голосует за всех троих. Это создаёт цепочки и потенциально — «суперголосователей» с огромным весом, что ведёт к концентрации власти.

**Наша система** ограничивает делегирование **одним уровнем**. Борис не может передать голос Анны дальше. Это принципиальное ограничение, предотвращающее неконтролируемое накопление влияния.

### 2. Весовые коэффициенты компетенций

| Аспект                      | Liquid Democracy             | Наша система                           |
| --------------------------- | ---------------------------- | -------------------------------------- |
| Вес голоса                  | Равный (1 человек = 1 голос) | С учётом компетенций                   |
| Профессиональная экспертиза | Не учитывается               | Учитывается через весовые коэффициенты |
| Peer-based оценка           | Нет                          | Да (2 очка/год)                        |

В Liquid Democracy все голоса равны — разница только в количестве делегированных голосов. Наша система вводит **[систему весов на основе профессиональных компетенций](../competency-weights/overview.md)**: голос генетика при голосовании по закону о ГМО весит больше, чем голос неспециалиста. Компетенции определяются через **[peer-based систему](../competency-weights/assignment.md)**, где профессионалы оценивают друг друга.

Этого механизма в Liquid Democracy **нет вообще**.

### 3. Инерционность (накопительная система)

| Аспект                         | Liquid Democracy | Наша система     |
| ------------------------------ | ---------------- | ---------------- |
| Защита от импульсивных решений | Нет встроенной   | Накопительные ИО |
| Скорость принятия решений      | Мгновенная       | Постепенная      |
| Фильтрация «шума»              | Нет              | Да               |

**Liquid Democracy** принимает решения по факту подсчёта голосов в определённый момент. Нет встроенного механизма защиты от импульсивных или эмоциональных решений.

**Наша система** использует **[накопительные избирательные очки (ИО)](../voting-process/inertia.md)**. Для смены власти или принятия закона необходимо устойчивое преимущество на протяжении длительного времени (недели, месяцы). Это работает как **фильтр низких частот** — пропускает устойчивые тренды и отсекает кратковременные всплески.

### 4. Блокчейн и криптография

| Аспект            | Liquid Democracy      | Наша система               |
| ----------------- | --------------------- | -------------------------- |
| Технология        | Не предписана         | Блокчейн + GPG             |
| Верификация       | Зависит от реализации | Криптографическая          |
| Открытость данных | Зависит от реализации | Абсолютная                 |
| Перевыпуск ключей | Не применимо          | Бесплатный, в любой момент |

Liquid Democracy — это концептуальная модель, не предписывающая конкретную технологию. Существующие реализации (LiquidFeedback, Decidim) работают на обычных серверах с централизованной базой данных.

В нашей системе **[блокчейн и GPG-ключи](../voting-process/blockchain.md)** — обязательная часть архитектуры. Каждый гражданин генерирует криптоключи, голоса публикуются в открытой распределённой сети, верификация математическая. Принцип «don't trust — verify» встроен в основу.

### 5. Непрерывность голосования

| Аспект                 | Liquid Democracy       | Наша система   |
| ---------------------- | ---------------------- | -------------- |
| Когда можно голосовать | Когда вопрос поставлен | В любой момент |
| Изменение голоса       | Обычно до дедлайна     | В любой момент |
| Публикация рейтингов   | По завершении          | Ежедневно      |
| Обратная связь         | Периодическая          | Постоянная     |

В Liquid Democracy голосование обычно происходит **по конкретным вопросам**, когда они возникают, с определённым дедлайном.

Наша система обеспечивает **[непрерывное голосование](../voting-process/continuous-voting.md)** — гражданин может изменить свой голос в любой момент, рейтинги публикуются ежедневно. Это создаёт **постоянную обратную связь** между обществом и властью, ликвидируя асимметрию влияния.

### 6. Назначение компетенций

| Аспект              | Liquid Democracy | Наша система               |
| ------------------- | ---------------- | -------------------------- |
| Понятие компетенций | Нет              | Да                         |
| Определение уровня  | Не применимо     | Peer-based + дипломы       |
| Прозрачность        | Не применимо     | Полная (кто кому дал очки) |

В Liquid Democracy нет понятия профессиональных компетенций. Наша система включает уникальный механизм **[peer-based назначения компетенций](../competency-weights/assignment.md)**: профессионалы ежегодно распределяют 2 очка среди коллег, формируя карту компетенций. Вход в систему — через диплом, дальнейшее развитие — через оценку сообщества.

## Резюме

**Liquid Democracy** — это модель **делегирования голосов**, элегантная в своей простоте.

**Непрерывная Прямая Демократия** — это **комплексная политическая система**, которая включает делегирование как один из компонентов, но существенно расширяет его:

- **Инерционностью** — защитой от манипуляций и импульсивных решений
- **Весовыми коэффициентами** — учётом профессиональных компетенций
- **Криптографической прозрачностью** — математической верификацией каждого голоса
- **Непрерывностью** — постоянной обратной связью между обществом и властью
- **Ограничением цепочек** — предотвращением концентрации влияния
