# Открытость и верификация

## Принцип абсолютной прозрачности

Предлагаемая система основана на фундаментальном принципе: **процесс голосования и валидации должен быть абсолютно открытым**. Каждый гражданин, каждая организация, каждый наблюдатель может в любой момент проверить любой аспект работы системы.

Это не просто декларация — прозрачность обеспечивается **технически** через использование блокчейна и криптографии.

## Что открыто

### Данные голосования

- Все голоса публикуются в открытой блокчейн-сети
- Каждый голос содержит криптографическую подпись
- Итоги подсчёта доступны в реальном времени
- История изменений голосов сохраняется полностью

### Законодательный процесс

- Текст каждого предложенного закона
- Результаты технической проверки
- Уровень поддержки инициатив
- Все правки с указанием авторства
- Итоги голосования по каждому закону

### Система компетенций

- Кто кому передал очки компетенций
- Уровень компетенции каждого профессионала
- Образовательный статус участников
- Весовые коэффициенты при голосовании

### Делегирование

- Факт делегирования голоса (при открытом режиме)
- Все override-операции
- История делегирований и отзывов

## Верификация голоса

### Личная проверка

Каждый гражданин может проверить свой голос:

1. **Найти свою запись** — по публичному ключу найти все свои голоса в блокчейне
2. **Проверить подпись** — убедиться, что подпись валидна и соответствует его ключу
3. **Проверить содержание** — убедиться, что голос записан так, как он его отдал
4. **Проверить учёт** — убедиться, что голос включён в итоговый подсчёт с правильным весом

### Делегирование проверки

Гражданин может делегировать право проверки **сторонним лицам**, не передавая свой приватный ключ:

**Способы делегирования:**

1. **Раскрытие публичного ключа** — гражданин сообщает доверенному лицу свой публичный ключ, и то может найти и проверить голоса в блокчейне
2. **Подписанное разрешение** — гражданин выдаёт подписанный документ, подтверждающий право на аудит его записей
3. **Указание ожидаемого голоса** — гражданин сообщает, за какой вариант он голосовал, и доверенное лицо проверяет соответствие

**Важно:** приватный ключ **никогда не передаётся**. Делегирование проверки — это право просмотра и верификации, а не право голоса.

## Общественный аудит

### Независимый подсчёт

Поскольку все данные открыты, **любой** может провести независимый подсчёт итогов:

```
1. Загрузить все записи голосования из блокчейна
2. Проверить валидность каждой криптографической подписи
3. Убедиться в отсутствии дублей (один ключ — один голос)
4. Применить весовые коэффициенты
5. Подсчитать итоговый результат
```

Результат должен совпасть с официальным. Если не совпадает — это сигнал о проблеме.

### Автоматизированный аудит

Предполагается создание **открытого программного обеспечения** для автоматической проверки:

- Скрипты для загрузки и верификации данных
- Инструменты для визуализации результатов
- Системы мониторинга аномалий
- API для интеграции с общественными наблюдательными организациями

Код этих инструментов также должен быть **открытым** (open source), чтобы каждый мог убедиться в их корректности.

## Защита от манипуляций

### Неизменяемость данных

Блокчейн обеспечивает **неизменяемость** записей:

- Записи нельзя удалить
- Записи нельзя изменить задним числом
- Любая попытка изменения будет обнаружена
- Целостность цепочки блоков математически гарантирована

### Распределённость

Нет **единой точки контроля**:

- Данные хранятся на множестве независимых узлов
- Ни один оператор не может контролировать всю сеть
- Для фальсификации необходимо скомпрометировать большинство узлов одновременно

### Криптографическая защита

- Каждый голос подписан приватным ключом
- Подделать подпись невозможно без приватного ключа
- Валидность подписи может проверить любой

## Уровни доверия

Система построена так, что **доверие не требуется** — вместо него используется **верификация**:

| Традиционная система            | Предлагаемая система          |
| ------------------------------- | ----------------------------- |
| Доверие избирательным комиссиям | Криптографическая верификация |
| Доверие наблюдателям            | Открытые данные для всех      |
| Доверие подсчёту                | Независимый пересчёт          |
| Доверие хранению бюллетеней     | Неизменяемый блокчейн         |
| «Мы верим»                      | «Мы проверяем»                |

## Принцип «не доверяй — проверяй»

В основе системы лежит криптографический принцип **«don't trust — verify»**:

- Не нужно **верить**, что голос учтён — можно **проверить**
- Не нужно **верить**, что подсчёт правильный — можно **пересчитать**
- Не нужно **верить**, что данные не изменены — можно **верифицировать** целостность блокчейна
- Не нужно **верить** чиновникам — можно **аудировать** систему

Это принципиально меняет отношения между обществом и властью: из модели доверия к модели прозрачности и проверяемости.
