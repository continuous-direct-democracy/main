# Жизненный цикл закона

## Принцип

В предлагаемой системе **ни один орган власти не может заблокировать закон**, предложенный гражданами. Процесс от идеи до принятия полностью открыт, а фильтрация происходит через **интерес общества** (порог поддержки), а не через решения чиновников. Весь процесс фиксируется в блокчейне.

## Этапы жизненного цикла

### 1. Инициация (свободная публикация)

Любой гражданин может опубликовать **идею закона** в свободной форме.

**Что требуется:**

- Заголовок предложения
- Краткое описание сути
- Текст предложения (может быть в свободной форме — от абстрактной идеи до детальной формулировки)
- Криптографическая подпись автора (GPG)

**Что НЕ требуется:**

- Одобрение какого-либо органа
- Юридически выверенная формулировка
- Соответствие каким-либо шаблонам

**Автоматические проверки** при публикации:

- ✅ Наличие обязательных полей (заголовок, описание, текст, подпись) — **блокирующая**
- ✅ Проверка на дубликаты (индекс уникальности) — **рекомендательная** (см. ниже)
- ✅ Подсказки по оформлению и юридической корректности — **рекомендательные**
- ✅ Проверка на конфликты — **рекомендательные**
- ❌ Проверка содержания — **не проводится**

:::note Статус предложения
До достижения порога поддержки публикация является **предложением** (proposal), а не законом. Предложения можно свободно игнорировать — они не отвлекают от основного процесса голосования.
:::

Подача фиксируется как транзакция в блокчейне с временной меткой.

### 2. Совместная работа и правки

После публикации предложение переходит в **фазу открытой разработки**.

**Механика:**

- Любой гражданин может предложить правки к тексту (аналогично pull requests в Git)
- Автор и сообщество обсуждают и принимают правки
- **Полная история версий** — всегда доступен текущий текст закона и все предыдущие версии
- Каждая правка фиксируется в блокчейне с указанием автора и временной метки

**Автоматические подсказки (рекомендательные):**

- Юридическое оформление — система подсказывает, как лучше сформулировать
- Стилистика — единообразие формулировок
- Возможные противоречия с действующими законами

**Важно:** автоматические подсказки **не могут заблокировать** развитие предложения. Они носят исключительно рекомендательный характер.

### 3. Набор поддержки

Для выхода на голосование предложение должно набрать **поддержку определённого числа граждан**.

**Параметры:**

- Порог поддержки: например, **5–10%** от общего числа избирателей
- Каждый избиратель может поддерживать одновременно ограниченное число предложений (например, **3**)
- Поддержка собирается **непрерывно** — гражданин может поддержать предложение в любой момент
- Если поддержка значительно превышает порог, период ожидания может быть сокращён

**Ключевое правило:** порог поддержки — это фильтр **релевантности**, а не цензуры. Система спрашивает: «Достаточно ли людям интересна эта тема?» Выборный орган **не имеет права заблокировать** предложение, набравшее порог поддержки.

Вся информация о текущем уровне поддержки публикуется ежедневно.

### 4. Непрерывное голосование

После набора поддержки предложение становится **вопросом для решающего голосования**.

**Механика:**

1. Каждый гражданин может проголосовать «за» или «против» в любой момент
2. Голосование происходит непрерывно
3. Голоса взвешиваются с учётом [компетенционных весов](../competency-weights/overview.md)
4. Когда проголосует достаточная доля электората (например, **60%** с учётом весов), запускается накопительный подсчёт
5. Ежедневно считается разница между взвешенными голосами «за» и «против»
6. Разница накапливается до достижения порога (например, **1850 ИО**)

### 5. Принятие или отклонение

- **Принятие:** накопленные ИО «за» достигают порога → закон принят
- **Отклонение:** накопленные ИО «за» падают до 0 → закон отклонён

**Ограничения по времени нет.** Голосование продолжается непрерывно до достижения одного из двух исходов. Если общество не определилось — это нормальное состояние, закон остаётся на голосовании, и граждане продолжают менять свои голоса. Механизм инерционности обеспечивает, что решение будет принято только тогда, когда общество сформирует устойчивую позицию.

:::tip Конфликты с действующими законами
Если принятый закон конфликтует с уже действующими законами, конфликт должен быть разрешён до вступления в силу. Подробнее — в разделе [Разрешение конфликтов](./conflict-resolution.md).
:::

### 6. Правки после принятия

После принятия закона граждане могут инициировать **правки** к нему по той же процедуре:

1. Правка подаётся как отдельное предложение со ссылкой на действующий закон
2. Проходит те же этапы: публикация → совместная работа → набор поддержки → голосование
3. При принятии правки закон обновляется
4. Вся история изменений сохраняется в блокчейне

## Индекс уникальности и работа с дубликатами

При публикации нового предложения автоматическая система рассчитывает **индекс уникальности** — степень сходства с существующими и ранее отклонёнными предложениями.

### Сценарии

| Индекс уникальности | Действие                                                         |
| ------------------- | ---------------------------------------------------------------- |
| Высокий (более 80%) | Публикация без замечаний                                         |
| Средний (40–80%)    | Рекомендация: «Похожие предложения уже существуют» со ссылками   |
| Низкий (менее 40%)  | Рекомендация оформить как правку к существующему закону          |
| Полный дубликат     | Автоматическое отклонение с предложением поддержать существующее |

### Если предложение похоже на ранее отклонённое

Если новое предложение содержит элементы ранее отклонённого закона, но с изменениями:

- Система **рекомендует** оформить его как правку к существующему закону
- Автор может **опубликовать как новое предложение**, указав ссылки на дубликаты и описав отличия
- Индекс уникальности публикуется вместе с предложением, чтобы граждане могли учитывать контекст при решении о поддержке

## Блокчейн-фиксация

На каждом этапе жизненного цикла в блокчейн записывается:

| Этап                  | Фиксируемые данные                                 |
| --------------------- | -------------------------------------------------- |
| Инициация             | Текст предложения, автор, временная метка, подпись |
| Правки                | Каждая правка с авторством и временной меткой      |
| Поддержка             | Каждый голос поддержки с подписью                  |
| Голосование           | Каждый голос «за» / «против» с подписью и весом    |
| Результат             | Итоговый статус (принят / отклонён)                |
| Правки после принятия | Все последующие изменения                          |

## Визуализация процесса

```
Идея → Публикация → Совместная работа → Набор поддержки → Голосование → Принятие
  │         │              │                   │               │            │
  ▼         ▼              ▼                   ▼               ▼            ▼
[блокчейн] [блокчейн]  [блокчейн]         [блокчейн]     [блокчейн]  [блокчейн]
```

Каждый шаг прозрачен и верифицируем. Ни один этап не может быть пропущен или подделан. **Ни один орган власти не может заблокировать предложение** на любом этапе — фильтрация происходит исключительно через волеизъявление граждан.
