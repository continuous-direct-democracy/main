# Разрешение конфликтов между законами

## Проблема

В системе, где законы принимаются непрерывно разными группами граждан, неизбежно возникают ситуации, когда **два закона определяют одни и те же сущности с разными значениями**. Например:

- Действующий закон устанавливает импортную пошлину на ввозимый товар в размере 50%
- Новый закон предлагает импортную пошлину в размере 10%

Такой конфликт должен быть обнаружен и разрешён **до вступления нового закона в силу**.

## Принцип

**Два разных закона не могут определять одни и те же сущности без указания приоритета.** Если обнаружен конфликт — он должен быть разрешён одним из установленных способов.

## Автоматическое обнаружение конфликтов

Система автоматически анализирует каждое предложение и действующий закон на предмет конфликтов:

- **На этапе публикации** — автоматика информирует автора о потенциальных конфликтах (рекомендательно, не блокирует публикацию)
- **На этапе набора поддержки** — информация о конфликтах публикуется вместе с предложением, чтобы граждане учитывали контекст
- **При принятии** — закон **не может вступить в силу** с неразрешёнными конфликтами

:::warning Важно
Конфликты **не блокируют** публикацию и голосование — они блокируют только **вступление в силу**. Это даёт сообществу время для обсуждения и выбора способа разрешения в ходе совместной работы над законом.
:::

## Способы разрешения

При обнаружении конфликта автор и сообщество должны выбрать один из путей:

### 1. Правка к существующему закону

Внести изменения в действующий закон по конфликтующему пункту.

**Механика:**

- Подаётся как отдельное предложение-правка со ссылкой на действующий закон
- Проходит стандартную процедуру: публикация → совместная работа → набор поддержки → голосование
- Оба предложения (новый закон и правка к старому) могут голосоваться параллельно
- Новый закон вступает в силу только после принятия правки к старому

**Пример:** Новый закон предлагает пошлину 10%. Параллельно подаётся правка к старому закону, убирающая пункт о пошлине 50%. Оба голосуются. Если оба приняты — конфликт разрешён.

### 2. Отмена старого закона

Инициировать полную отмену действующего закона.

**Механика:**

- Подаётся предложение об отмене старого закона целиком
- Проходит стандартную процедуру голосования
- Новый закон вступает в силу только после отмены старого

**Когда это уместно:** когда старый закон полностью устарел и новый закон является его комплексной заменой.

### 3. Исключение конфликтующего пункта

Убрать спорный пункт из нового закона.

**Механика:**

- Автор или сообщество вносят правку в новый закон, удаляя конфликтующий пункт
- Правка проходит через стандартный процесс совместной работы
- После удаления конфликтующего пункта закон может вступить в силу

**Когда это уместно:** когда конфликтующий пункт не является ключевым для нового закона.

### 4. Явное указание приоритета

Если законы находятся на **разных уровнях** иерархии, конфликт может быть разрешён через указание приоритета.

**Механика:**

- Новый закон явно указывает, что он имеет приоритет над конкретным пунктом старого закона
- Приоритет определяется в соответствии с действующей иерархией (см. ниже)
- Если приоритет нового закона выше — конфликт разрешён автоматически

## Иерархия приоритетов

Законы имеют уровни приоритета в зависимости от области действия:

| Уровень         | Приоритет | Пример                   |
| --------------- | --------- | ------------------------ |
| Конституционный | Высший    | Основные права и свободы |
| Федеральный     | Высокий   | Общенациональные законы  |
| Региональный    | Средний   | Законы субъекта/региона  |
| Муниципальный   | Базовый   | Местные постановления    |

**Правила приоритета:**

- Закон более высокого уровня имеет приоритет над законом более низкого уровня
- Законы одного уровня **не могут конфликтовать** — конфликт должен быть разрешён одним из способов выше
- Действующая иерархия приоритетов **сама является предметом голосования** и может быть изменена через стандартную процедуру

:::note Самоизменяемость
Правила иерархии и разрешения конфликтов — это **начальные параметры** системы. Они могут быть пересмотрены через тот же механизм непрерывного голосования (см. [Заключение](../conclusion.md)).
:::

## Процесс разрешения

```
Конфликт обнаружен
       │
       ▼
┌──────────────────┐
│ Законы на разных │──── Да ──→ Приоритет по иерархии
│ уровнях?         │
└──────────────────┘
       │ Нет
       ▼
┌──────────────────────────────────────┐
│ Выбор способа разрешения:            │
│                                      │
│ 1. Правка к существующему закону     │
│ 2. Отмена старого закона             │
│ 3. Исключение пункта из нового       │
└──────────────────────────────────────┘
       │
       ▼
  Голосование по выбранному способу
       │
       ▼
  Конфликт разрешён → Новый закон вступает в силу
```

## Что делать, если конфликт не разрешён

Если сообщество не может прийти к решению по способу разрешения конфликта:

- Новый закон **остаётся на голосовании**, но не вступает в силу
- Граждане продолжают обсуждать и предлагать правки
- Ограничения по времени нет — конфликт разрешается тогда, когда общество готово
- Автоматика продолжает уведомлять о неразрешённом конфликте
